# ToDo List - Приложение для управления задачами

## Описание проекта

Веб-приложение для управления задачами (ToDo List) на Django. Позволяет создавать, редактировать, удалять задачи и переключать их статусы.

## Структура проекта

- `todo/app/models.py` - модель Task с полями: title, status, created_at
- `todo/app/views.py` - представления для CRUD операций
- `todo/app/templates/` - HTML шаблоны
- `todo/app/static/style/style.css` - стили приложения
- `todo/app/tests.py` - unit тесты

## Запуск проекта

```bash
# Установка виртуального окружения в корневую папку проекта (2-todo)
py -m venv venv

# Активация виртуального окружения
venv\Scripts\activate

# Установка зависимостей 
pip install django

# СОздание миграций
py todo/manage.py makemigrations

# Применение миграций
py todo/manage.py migrate

# Запуск сервера разработки
py todo/manage.py runserver
```

## Запуск тестов

```bash
cd todo
py manage.py test app # (с запущеным сервером разработки в другой консоли)
```

## Описание Unit тестов

### Тест 1: Создание задачи с дефолтными значениями
**Класс:** `TaskModelTest.test_task_creation`  
**Сценарий:**
1. Создается новая задача с указанием только названия
2. Проверяется, что задача создана с корректным названием
3. Проверяется, что статус по умолчанию установлен как "TODO" (внутренний объект Status)
4. Проверяется, что дата создания автоматически установлена
5. Проверяется строковое представление объекта (метод `__str__`)

**Ожидаемый результат:** Задача создается с дефолтными значениями (статус TODO, автоматическая дата создания)

---

### Тест 2: Проверка сортировки задач по дате создания
**Класс:** `TaskCollectionTest.test_task_list_ordering`  
**Сценарий:**
1. Создаются три задачи последовательно (task1, task2, task3)
2. Получается коллекция всех задач через QuerySet, отсортированная по дате создания (убывание)
3. Коллекция преобразуется в список для проверки порядка
4. Проверяется порядок задач в коллекции
5. Проверяется, что последняя созданная задача находится первой в списке

**Ожидаемый результат:** Задачи отсортированы по дате создания в обратном порядке (новые первыми)

---

### Тест 3: Фильтрация задач по статусу
**Класс:** `TaskCollectionTest.test_task_filtering_by_status`  
**Сценарий:**
1. Создаются три задачи с разными статусами (TODO, IN_PROGRESS, DONE)
2. Выполняется фильтрация коллекции задач по каждому статусу через QuerySet.filter()
3. Проверяется количество задач для каждого статуса через метод count()
4. Проверяется, что каждая задача находится в правильной отфильтрованной коллекции через assertIn()

**Ожидаемый результат:** Фильтрация работает корректно, каждая задача попадает в правильную коллекцию по статусу

---

### Тест 4: Циклическое переключение статуса через все состояния
**Класс:** `TaskStatusToggleTest.test_status_toggle_cycle`  
**Сценарий:**
1. Создается задача со статусом TODO
2. Используется внутренний объект order (список статусов) для циклического переключения
3. Статус переключается на IN_PROGRESS (первое переключение) через вычисление индекса
4. Статус переключается на DONE (второе переключение)
5. Статус переключается обратно на TODO (третье переключение, цикл замыкается)
6. На каждом шаге проверяется корректность нового статуса

**Ожидаемый результат:** Статус циклически переключается: TODO → IN_PROGRESS → DONE → TODO

---

### Тест 5: Валидация формы с корректными данными
**Класс:** `TaskFormTest.test_form_valid_data`  
**Сценарий:**
1. Создаются данные формы с корректным названием и статусом
2. Создается экземпляр формы TaskForm с этими данными
3. Проверяется валидность формы через метод is_valid()
4. Форма сохраняется, создавая новую задачу через метод save()
5. Проверяется, что задача создана с правильными данными

**Ожидаемый результат:** Форма валидна, задача успешно создается

---

### Тест 6: POST запрос на создание задачи через view
**Класс:** `TaskViewsTest.test_task_create_view_post`  
**Сценарий:**
1. Подготавливаются данные для новой задачи
2. Выполняется POST запрос на создание задачи через Django test client
3. Проверяется статус ответа (302 - редирект)
4. Проверяется, что задача создана в базе данных через QuerySet.filter().exists()

**Ожидаемый результат:** Задача успешно создается, происходит редирект на список задач

---

## Покрытие функционала

Тесты покрывают:
- Создание объектов модели Task
- Работу с коллекциями задач (сортировка, фильтрация)
- Переключение статусов (циклическое)
- Валидацию форм
- CRUD операции через представления
- Внутренние объекты (Status choices, методы модели)
- Работу с QuerySet и коллекциями Django ORM
